# 詳細設計

## Phase 1 — フロントエンドUI（`phase1_ui/`）

### ファイル構成

```
phase1_ui/
├── index.html   レイアウト・DOM骨格
├── style.css    ダークテーマ
└── app.js       状態管理・全ロジック
```

ビルドなし・依存ライブラリなし。`index.html` をブラウザで直接開いて動作する。

---

### アプリ状態（`app.js`）

```js
const state = {
  layout: [],              // 2D配列: null=通路 | 座席ID文字列
  seats: {},               // { [seatId]: { categoryId, ticketNo } }
  activeCategoryId: 'A',  // 現在選択中の区分ID
};
```

`layout` と `seats` は独立しており、`layout` は物理配置のみ、`seats` は割当状態のみを持つ。
この分離により「通路定義」と「区分割当」を混在させない。

---

### 座席区分（`CATEGORIES`）

`app.js` 冒頭の定数配列。区分の追加・変更はここだけ編集する。

```js
const CATEGORIES = [
  { id: 'A',     name: 'A区分',   color: '#e74c3c' },
  { id: 'B',     name: 'B区分',   color: '#3498db' },
  { id: 'C',     name: 'C区分',   color: '#2ecc71' },
  { id: 'D',     name: 'D区分',   color: '#f39c12' },
  { id: 'VIP',   name: 'VIP',     color: '#9b59b6' },
  { id: 'STAFF', name: 'スタッフ', color: '#1abc9c' },
];
```

> **注意**: `id` はJSONキーになるため、既存データがある状態で変更すると互換性が壊れる。

---

### 主要関数

| 関数 | 引数 | 説明 |
|------|------|------|
| `buildDefaultLayout()` | なし | 仮レイアウト（10行×12列、列3・8が通路）を生成。Excel実装後は削除予定 |
| `renderGrid()` | なし | `state.layout` からグリッドDOMを全再構築。`applySeatStyle` を各座席に適用 |
| `applySeatStyle(cell, seatId)` | DOM要素, 座席ID | `state.seats[seatId]` の状態に応じてセルの色・テキストを更新 |
| `onSeatClick(seatId, cell)` | 座席ID, DOM要素 | クリック時: `activeCategoryId` を `state.seats` に書き込み → `applySeatStyle` |
| `showContextMenu(x, y, seatId)` | 座標, 座席ID | 右クリックメニューを表示。区分変更・消去のオプションを動的生成 |
| `renderSummary()` | なし | `state.seats` を集計してサイドバーのカテゴリ別件数・合計を更新 |
| `assignTickets()` | なし | `layout` の行列順（上→下、左→右）に `{prefix}{3桁連番}` を採番 |
| `exportJSON()` | なし | `state` + `ticketConfig` をJSONファイルとしてダウンロード |
| `importJSON(file)` | Fileオブジェクト | JSONを読み込み `state.layout` / `state.seats` を上書き後、`renderGrid()` |

### UIレイアウト

```
┌─────────────────────────────────────────────────┐
│ ヘッダー: タイトル | 区分ボタン | アクションボタン │
├──────────────────────────────────┬──────────────┤
│                                  │ 集計パネル   │
│  【ステージ】                     │ (区分別件数) │
│                                  │              │
│  座席グリッド                     │ チケット     │
│  (行ラベル + セル)                │ 採番パネル   │
│                                  │              │
└──────────────────────────────────┴──────────────┘
```

グリッドは `CSS Grid` で描画。列数は `state.layout[0].length` から動的に決まる。

---

### 座席ID命名規則（現状・仮）

```
行ラベル(A-J) + 列番号(通路を除いてカウント)
例: A1, A2, A3, A4 ... (列3・8が通路のため列番号は通路をスキップ)
```

Excel実装後、このルールはExcel側の定義に準拠させる。

### Python子プロセス管理（`main.js`）

```js
// Pythonサーバーをアプリ起動時に子プロセスとして起動
const { spawn } = require('child_process');
const pyServer = spawn('python', ['../phase2_python/server.py']);

// アプリ終了時に確実にkill
app.on('before-quit', () => pyServer.kill());
```

### BrowserWindow設定

`phase1_ui/index.html` をそのままロードする。
Excelインポート・出力ボタンのみ、`preload.js` 経由で `ipcRenderer` を使いPython APIを呼ぶ追加実装が必要。
