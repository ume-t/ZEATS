# Excelフォーマット解析

対象ファイル：`大阪フォーマット.xlsx`（`tmp/` 配下、Git管理外）

---

## シート構成

| シート名 | 最大行 | 最大列 | 用途 |
|---------|--------|--------|------|
| 全体図 | 48 | 158 | 全エリアの俯瞰図（参照用） |
| マーブルビーチサイド | 69 | 803 | 座席割当シート |
| アウトレットサイド | 40 | 891 | 座席割当シート |
| テーブル席 | 100 | 88 | 座席割当シート |
| テーブル席（追加） | 23 | 47 | 座席割当シート |

インポート対象は「全体図」を除く4シート。

---

## 座席シートの共通レイアウト

```
A2          : イベント名（例: '大阪芸術花火2026'）
A3 or A4    : エリア名（シート名と同じ）
A列（凡例行）: 販売チャネル名
B列（凡例行）: チャネルごとの総枚数
C列         : 列ラベル（'1列', '2列', ...）
座席セル    : 値=座席番号（整数）、背景色=チャネルカラー
```

### 凡例の位置

- マーブルビーチサイド：行17〜24
- その他のシートも同様の構造（行位置はシートにより異なる）

### ブロック構造

- 行方向：セクションヘッダー行（例: 'エキサイティングシート', 'ビーチイス席（指定）'）の下にブロックが続く
- 列方向：ブロックヘッダー（例: '赤ゾーン31ブロック'）がある列から座席が右方向に並ぶ
- ブロック間は空列で区切られる

## 座席IDの命名規則

アプリ内での座席の一意識別子を以下の形式で構成する。

```
{シート名}_{ブロック名}_{列番号}列_{座席番号}番
例: マーブル_赤31_1列_10番
```

シート名は短縮形を使用（パース時にマッピングで変換）。

---

## インポート時のパース方針

1. 凡例行を走査し、チャネル名→色コードの辞書を生成
2. セクションヘッダー行（`値が文字列かつ列Cが空`）を検出
3. ブロックヘッダー行（`'ブロック'を含む文字列のセル`）を検出し、開始列を記録
4. 列Cの `'N列'` パターンで列番号を取得
5. 座席セル（値が整数）を走査し、色コードからチャネルを逆引き
6. 結果を `layout`（2D配列）と `seats`（チャネル割当辞書）に変換
